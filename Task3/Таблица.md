# Таблица классификации и защиты данных

| Данные | Класс / Состояние | Необходимость защиты | Инструменты защиты (вкл. автоконтроль) |
|--------|-------------------|----------------------|----------------------------------------|
| **ПДн базовые**<br>(ФИО, ДР, телефон, email, адрес) | `Confidential`<br>`at rest / in transit / in use` | Требования 152‑ФЗ,<br>предотвращение кражи личности,<br>минимизация доступа | Шифрование БД; TLS;<br>RBAC + ABAC / RLS;<br>маскирование в логах и непроде;<br>DLP для почты;<br>аудит доступа (SIEM) |
| **Медицинские данные**<br>(анализы, диагнозы, назначения, медкарта) | `Restricted / PHI`<br>`at rest / in transit` | Медтайна (323‑ФЗ),<br>высокий ущерб при утечке | Выделенный домен;<br>шифрование; mTLS;<br>псевдонимизация (order_id);<br>TBAC по тегу `PHI`;<br>presigned URL;<br>антивирус при загрузке;<br>неизменяемый аудит (WORM) |
| **Документы / сканы**<br>(паспорт, полис, договоры, согласия) | `Confidential`<br>`файлы at rest / in transit` | Содержат ПДн и реквизиты;<br>часто рассылаются | Объектное хранилище с SSE‑KMS;<br>Object Lock (WORM);<br>теги `no_email` / `confidential`;<br>DLP; водяные знаки;<br>ссылки с коротким TTL;<br>журнал доступа |
| **Платёжные данные**<br>(чеки, суммы, receipt_id; без PAN) | `Internal / Confidential`<br>`at rest / in transit` | Целостность учёта;<br>запрет хранения карт | TLS;<br>не хранить PAN;<br>шифрование реестров;<br>разделение ролей (кассир/бухгалтер);<br>аудит операций;<br>шифрованные бэкапы |
| **Секреты / ключи / токены**<br>(пароли, API‑ключи, сессии) | `Restricted`<br>`at rest / in transit` | Компрометация = полный доступ | Secrets Vault / KMS;<br>ротация ключей / TTL;<br>mTLS;<br>запрет в коде и файлах;<br>автоскан секретов в CI;<br>аудит выдачи |
| **Резервные копии / дампы / снапшоты** | `Confidential / Restricted`<br>`at rest` | Наибольший масштаб утечки | Шифрование бэкапов;<br>раздельные учётки;<br>Object Lock / immutability;<br>регулярные тесты восстановления;<br>мониторинг копий |
| **Логи / трейсы** | `Internal`<br>(могут содержать ПДн)<br>`at rest / in transit` | Утечки ПДн через логи | Маскировка / редакция ПДн;<br>централизованное хранилище;<br>ограничение доступа;<br>ретеншн;<br>корреляция в SIEM + алерты |
| **Аналитические датасеты**<br>(BI / ML) | `Internal`<br>`at rest / in use` | Риск повторной идентификации | Обезличивание / агрегация;<br>k‑анонимность;<br>токенизация;<br>запрет прод‑ПДн в непрод;<br>TBAC по тегам;<br>контроль выгрузок |
| **Трафик API и межсервисные вызовы** | `Transit` | Перехват / несанкционированный доступ | TLS 1.2+ / mTLS;<br>API‑шлюз + WAF;<br>валидация схем / скоупов;<br>rate‑limit;<br>OPA (политики);<br>автотесты контрактов и security‑сканы |
